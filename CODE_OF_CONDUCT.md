# Code of Conduct

## Our Standard

Leadping SDK projects should be practical, respectful, and focused on helping developers use the Leadping API.

Expected behavior includes:

- Use welcoming and professional language.
- Respect differing viewpoints and experience levels.
- Keep discussions focused on the issue, pull request, or SDK behavior.
- Accept constructive feedback gracefully.
- Report security issues privately through the process in `SECURITY.md`.

Unacceptable behavior includes:

- Harassment, threats, insults, or discriminatory language.
- Publishing private information without permission.
- Repeated off-topic disruption.
- Any conduct that would make the project unsafe or unprofessional.

## Enforcement

Maintainers may edit, hide, or remove comments, issues, pull requests, or other contributions that violate this code of conduct. Maintainers may also limit or block participation when needed to protect the project.

To report a conduct concern, email support@leadping.ai.
